🔍 Overview

Design patterns are reusable solutions to common problems in software design.
They help improve:

- Maintainability
- Flexibility
- Scalability

This guide focuses on:

- Low Coupling → reducing dependencies between components
- High Cohesion → keeping related functionality together

🏗️ Creational Patterns

1. Singleton (Creational)

Intent:
Ensures a class has only one instance and provides a global access point.

OOP Principle:
Encapsulation of instantiation logic

Analogy:
🏛️ The Government — A country has only one official government that everyone interacts with.

Best For:

- Database connections
- Configuration settings

🔄 Behavioral Patterns 2. Strategy (Behavioral)
Intent:
Defines a family of algorithms, encapsulates each one, and makes them interchangeable.

OOP Principle:
Composition over Inheritance

Analogy:
🗺️ A GPS App — You can switch between Drive, Walk, or Cycle modes while the app remains the same.

Best For:

- Payment systems (Credit Card vs PayPal)
- Sorting algorithms

6. Observer (Behavioral)
   Intent:
   Creates a subscription system to notify multiple objects when a change occurs.

OOP Principle:
Low Coupling using Interfaces

Analogy:
📺 YouTube Channel — Subscribers receive notifications when new content is posted.

Best For:

- Event handling
- Push notifications
- Real-time systems

🧩 Structural Patterns 3. Proxy (Structural)
Intent:
Provides a placeholder to control access to another object.

OOP Principle:
Interface implementation & method overriding

Analogy:
💳 Credit Card — Acts as a middleman between you and your bank account.

Best For:

- Access control
- Logging
- Lazy loading / caching

4. Bridge (Structural)
   Intent:
   Separates abstraction from implementation so both can evolve independently.

OOP Principle:
Low Coupling

Analogy:
📺 Remote Control — The same remote works with different devices (TV, Radio).

Best For:

- Cross-platform applications
- UI logic vs OS-specific rendering

5. Decorator (Structural)
   Intent:
   Adds new functionality dynamically without modifying existing code.

OOP Principle:
Open/Closed Principle

Analogy:
☕ Coffee Customization — Start with coffee, then add milk, sugar, or cream.

Best For:

- UI components
- File processing (encryption, compression)
